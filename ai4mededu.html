⚠️ 数据泄漏（Data Leakage）警告
你选中了一些"伪特征"——它们看似相关，实则是结果而非原因，或者来自未来信息。
例如"是否抗生素使用"——这本身就是医生已经判断为重症后的决策结果；用它来预测重症，等于"事后诸葛亮"。
在真实部署时，模型会彻底失效。
选择特征的过程，本质上是把医学知识编码进算法的过程。
有经验的医生选"月龄 + 体温 + WBC + CRP + 反应情况"——这不是数据驱动，这是临床驱动。

另一种路线是"端到端深度学习"：把所有原始数据扔给神经网络，让它自己发现模式。 这在影像识别中极其成功，但在儿科这种错误代价极高的领域，我们通常选择可解释性优先。
⚖️ 可解释性 vs 预测力

这是医学 AI 中永恒的张力：
· 可解释模型（逻辑回归、决策树）：医生看得懂每个变量的权重，能质疑、能负责。
· 黑箱模型（深度学习）：预测力可能更强，但医生无法解释"为什么这么判"。
在儿科，我们倾向前者。但"倾向"不等于"永远"——下一幕你会看到模型的家族谱。

⚠️ 警觉疲劳：AI 最大的非技术性失败

你刚才连续处理了 3 个患儿。注意观察你自己：到第 3 个时，你是否开始直接接受 AI 的建议？

这就是警觉疲劳（Alert Fatigue）——AI 用多了，医生反而失去了独立判断的警觉。 研究显示，在连续 5 次以上 AI 提示后，医生的独立核查率下降 40%。

这是医疗 AI 部署后最常见、也最危险的失败模式。一个好的 AI 不是让医生变"懒"，而是让医生在正确的地方"专注"。

📖 部署说明（教师必读）

方案 A · 本地直接打开（推荐开发测试）：
Chrome 启动添加参数 --disable-web-security --user-data-dir=/tmp/chrome，可绕过 CORS。仅用于教师本机。

方案 B · 自建简单代理（推荐课堂使用）：
用任意服务器（或 Cloudflare Worker）部署一个转发服务，将 /proxy?url=xxx 转发到目标 API。示例 Cloudflare Worker 代码：

export default {
  async fetch(req) {
    const u = new URL(req.url);
    const target = u.searchParams.get('url');
    if (!target) return new Response('need ?url=', {status:400});
    const r = await fetch(target, {
      method: req.method, headers: req.headers, body: req.body
    });
    const resp = new Response(r.body, r);
    resp.headers.set('Access-Control-Allow-Origin', '*');
    return resp;
  }
}
